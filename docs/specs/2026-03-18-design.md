# alpha-claude-telegram-ask-user -- Design Spec
Date: 2026-03-18

## 1. Purpose

A standalone npm package that bridges the Claude Agent SDK `ask_user` MCP tool with Telegram inline keyboards. When a Claude agent calls `ask_user`, the user receives a Telegram message with clickable buttons and the agent waits for the answer before continuing.

No other open source project implements this bridge.

## 2. Scope

**In scope:**
- MCP server exposing the `ask_user` tool (Claude Agent SDK)
- Telegram inline keyboard builder (single-select and multi-select)
- ForceReply handler for free-text answers ("Autre" option)
- Callback query handler that resolves the pending promise
- In-memory storage (default) with optional storage adapter interface
- Configurable timeout (default 5 minutes)
- grammY adapter (included)
- Telegraf adapter (included)

**Out of scope:**
- Discord, Slack, or other transports
- Persistent storage (delegated to optional adapter)
- Bot creation or management (user brings their own bot)

## 3. Architecture

```
User code
  |
  +--> createTelegramUiServer(adapter, chatId, options?)
  |      returns { server }  <-- pass to mcpServers in Claude query
  |
  +--> hook bot callback_query -> handleQuestionCallback(adapter, action, callbackQueryId)
  |
  +--> hook bot message:text  -> handleForceReply(questionId, text, adapter)
  |
  +--> setInterval(() -> cleanExpiredQuestions(adapter), 60_000)
```

### 3.1 TelegramAdapter interface

Abstracts grammY, Telegraf, or any future transport:

```ts
interface TelegramAdapter {
  sendMessage(chatId: string, text: string, keyboard?: InlineKeyboardMarkup): Promise<{ messageId: number }>
  editMessage(chatId: string, messageId: number, text: string, keyboard?: InlineKeyboardMarkup): Promise<void>
  answerCallbackQuery(callbackQueryId: string): Promise<void>
  sendForceReply(chatId: string, text: string): Promise<{ messageId: number }>
}
```

Included adapters:
- `createGrammyAdapter(bot: Bot)` -- wraps grammY
- `createTelegrafAdapter(bot: Telegraf)` -- wraps Telegraf

### 3.2 Storage interface

```ts
interface QuestionStore {
  set(id: string, question: PendingQuestion): void
  get(id: string): PendingQuestion | undefined
  delete(id: string): void
  getExpired(now: number): PendingQuestion[]
}
```

Default: `InMemoryStore` (built-in, no dependencies).
Optional: user injects their own (SQLite, Redis, etc.).

### 3.3 MCP Server

Created via `createTelegramUiServer(adapter, chatId, options?)`.

Options:
```ts
{
  timeoutMs?: number     // default: 300_000 (5 min)
  store?: QuestionStore  // default: InMemoryStore
}
```

Exposes one tool: `ask_user` with params:
- `question` (string) -- the question text
- `options` (array, 2-4 items) -- `{ label, description }`
- `header` (string, optional) -- short label shown in brackets
- `multi_select` (boolean, optional) -- default false

Returns the chosen label (string) or a comma-separated list for multi-select.

### 3.4 Pending resolvers

Module-level `Map<questionId, { resolve, reject }>` shared between:
- `createTelegramUiServer` (writes resolvers)
- `handleQuestionCallback` (resolves them)
- `cleanExpiredQuestions` (resolves with timeout sentinel)

### 3.5 ForceReply ("Autre")

Module-level `Map<messageId, questionId>` (`pendingForceReplies`) maps the bot's ForceReply message ID to the question. The user's reply resolves the promise with the free text.


## 6. Error handling

- Telegram API failure on `sendMessage`: return graceful error string to Claude ("Failed to reach user. Proceed with best judgment.")
- Timeout: edit message to show expiry, resolve promise with sentinel `__timeout__`, return timeout message to Claude
- Double-tap: check `answered` flag before resolving, silently ignore duplicate callbacks
- Unknown questionId in callback: log warning, silently ignore

## 7. Known bugs in current code (to fix)

4. All logic in one file -- split into focused modules per section 4

## 8. Success criteria

- `npm install alpha-claude-telegram-ask-user` works with zero mandatory peer deps
- grammY example runs end-to-end: agent asks question, user taps button, agent receives answer
- Telegraf example runs end-to-end
- Custom store example shows SQLite injection
- No circular imports, TypeScript strict mode passes
